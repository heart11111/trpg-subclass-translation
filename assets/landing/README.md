# 랜딩 페이지 — 시네마틱 스크롤 씬

`index.html` + `assets/landing.css` + `assets/landing.js` 로 구성된
스크롤 고정(sticky) 스테이지 하나가 전체 랜딩을 담당한다.
로컬 미리보기: 리포 루트에서 `python3 -m http.server` 후 `http://localhost:8000`.

## 타임라인 (정규화 진행도 p)

| p | 비트 |
|---|---|
| 0.00–0.03 | 히어로 홀드 (성채 + 타이틀) |
| 0.03–0.16 | 타이틀/인트로 퇴장 |
| 0.10–0.42 | 카메라 푸시인 + 전경 소나무 좌우 분리 |
| 0.21–0.42 | 내러티브 A "국경의 모험가들" (1부) |
| 0.36–0.46 | 성채 → 왈든 파노라마 크로스페이드 + 초점 회복 |
| 0.48–0.74 | 내러티브 B "크로스로드에서 만나요" (2부) |
| 0.75–1.00 | 자료 서고 가로 카탈로그 진입, 컨트롤 표시 |

수치는 `assets/landing.js` 의 `BEAT` 객체에서 조정한다.
JS는 CSS 커스텀 프로퍼티만 쓰고, 최종 transform은 CSS가 소유한다.
reduced-motion / no-JS 에서는 전 비트가 일반 흐름으로 세로 적층된다.

## 레이어 에셋 명세 (교체 지점)

| 역할 | 현재 파일 | 규격/앵커 | 교체 시 (ima2 생성) |
|---|---|---|---|
| 00 히어로 월드 플레이트 | `assets/images/cover-arrowkeep-v2.jpg` | 16:9+, 불투명, 중앙 40% 포커스 | 동일 구도의 밤 성채 일러스트. 좌하단 1/3 은 타이틀 자리로 비워둘 것 |
| 20 파노라마 플레이트 | `assets/landing/panorama-walden.webp` | 16:9, 불투명 | 2부 무대(크로스로드 항구도시) 전경이 이상적 |
| 40/41 전경 오클루더 | `assets/landing/fg-pines.svg` | 720×1080, 좌측 기준(우측은 CSS 미러), 하단 앵커, 밑변 솔리드 | 투명 PNG/WebP 소나무·바위 클러스터. 하단이 뚫려 있으면 배경이 비치니 주의 |
| 카탈로그 카드 6장 | `assets/landing/card-*.webp` | 760px 폭, 16:9 크롭 | 각 문서 대표 일러스트 |
| (선택) 하늘 루프 영상 | 없음 | 16:9 mp4/webm, 10–15s 무한루프 | `plate-keep` 아래에 `<video>` 레이어로 추가 가능. 구름 흐름/별 반짝임 정도의 미세 모션 |

에셋 파일명만 유지하면 코드 수정 없이 교체된다.
새 파일명을 쓸 경우 `landing.css` 의 `plate-keep` / `plate-pano` 배경 URL 과
`index.html` 카드 `<img>` 경로를 바꾼다.

## 미해결 / 로컬(ima2) 작업 목록

- 진짜 깊이 레이어 분리(하늘/원경/중경/성채 컷아웃)는 ima2 로 생성해 교체 예정.
  이 세션(클라우드)은 이미지 생성 결과물 다운로드가 네트워크 정책에 막혀 있었음.
- Grok 등으로 하늘 앰비언트 루프 영상을 만들면 히어로에 생동감 추가 가능.
- 카드 05/06(크로스로드/RP 로그)은 실사풍 아트라 챠비 톤과 스타일이 섞임 —
  통일하려면 ima2 로 챠비 톤 카드 아트 재생성.
